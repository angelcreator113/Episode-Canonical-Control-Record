KNOWN BEHAVIORS
───────────────────────────────────────────────────────────────────────────────

1. DELETE PERMISSION
   - Only admin users can delete episodes
   - Non-admin users will receive 403 Forbidden
   - This is by design for data safety
   - Soft delete preserves data for audit trails

2. PUBLIC GET ENDPOINTS
   - GET /api/v1/episodes (list) is publicly accessible
   - GET /api/v1/episodes/:id (detail) is publicly accessible
   - No authentication required for read operations
   - POST/PUT/DELETE require authentication

3. FIELD NAMING
   - API accepts both camelCase and snake_case
   - Database stores in snake_case
   - Frontend converts camelCase to snake_case
   - Example: episodeNumber → episode_number

4. TIMESTAMPS
   - created_at: Set at creation, never changed
   - updated_at: Updated on every modification
   - Both in ISO 8601 format (UTC)
